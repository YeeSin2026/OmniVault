# 视觉抓取方案说明

OmniVault 包含一套**可选的视觉采集方案**，通过截图 + 多模态视觉模型（VLM）理解页面内容。

---

## 核心理念

**登录你的个人账号后，视觉方案可以采集分享链接里的全部信息。**

默认方案（Playwright/httpx）只能抓取页面 HTML 中直接可见的文本和图片。但现代社媒平台大量内容是动态渲染、登录可见、或者藏在交互层里：

- **评论区** — 高价值讨论，Playwright 可能抓不到或需要额外 API 调用
- **弹幕** — B站等平台的弹幕数据在 DOM 之外
- **互动数据** — 点赞数、转发链、收藏夹
- **推荐流** — 相关内容的算法推荐
- **登录墙内容** — 小红书/公众号部分内容需登录态

视觉方案的优势：只要是你在浏览器里能看到的内容，就能截图 → VLM 识别 → AI 筛选高价值部分。

**不会一股脑全存。AI 会自动筛选高价值评论、过滤水军/表情包/无意义内容，只保留有信息量的部分。**

---

## 两种方案对比

| | 默认方案（Playwright/httpx） | 视觉方案（VLM） |
|------|------|------|
| 原理 | 模拟浏览器请求 / 操作 DOM | 截图 → 多模态大模型识别 |
| 采集范围 | HTML 文本 + 图片 | **页面可见的全部内容**（含评论/弹幕/互动数据） |
| 适用平台 | 所有平台 | 所有平台 |
| 依赖 | Docker 内置 Chromium | 本地多模态大模型（需自备） |
| 风险 | 低 | **可能触发平台风控，导致账号受限** |
| 默认状态 | ✅ 启用 | ❌ 关闭 |

---

## 启用视觉方案

在 `.env` 中设置：

```bash
VISION_ENABLED=true
```

## 模型要求

视觉方案需要**支持多模态视觉识别的大模型**，例如：

- Gemma 4（本地 OMLX）
- GPT-4V / GPT-4o（OpenAI）
- Qwen-VL（通义千问）
- Claude 4 Opus（Anthropic）
- UI-TARS（专为 GUI 理解训练）

配置方式：

```bash
# 本地模型（OMLX）
VISION_ENDPOINT=http://127.0.0.1:8000/v1/chat/completions
VISION_MODEL=UI-TARS-1.5-7B-6bit

# 或远程 API
VISION_ENDPOINT=https://api.openai.com/v1/chat/completions
VISION_MODEL=gpt-4o
VISION_API_KEY=sk-xxxx
```

## ⚠️ 风险告知

视觉方案通过操作系统级别的截图和模拟点击来获取内容。虽然从平台角度看更接近真人操作，但仍然存在以下风险：

1. **账号风控** — 平台可能检测异常操作频率并限制账号功能
2. **内容获取不稳定** — 依赖浏览器窗口状态，受弹窗/网络/页面改版影响
3. **仅支持 macOS** — 后台截图和点击功能依赖 macOS API（CGEventPostToPid）
4. **模型识别偏差** — VLM 可能遗漏或误解页面内容

**使用视觉方案导致的任何账号问题（包括但不限于封号、限流、功能受限），本项目不承担任何责任。请自行评估风险后使用。**

## 技术实现

代码位于 `src/visual/`，核心组件：

- **截图** — macOS `screencapture` 命令
- **定位** — AX API（原生 App）+ VLM 视觉检测（网页）
- **点击/输入** — Swift 原生二进制 `bgclick` / `bgtype`（CGEventPostToPid）
- **视觉理解** — UI-TARS / GPT-4V / Gemma 4 等多模态大模型

视觉方案为可选增强功能，**不会在默认流程中启用**。
